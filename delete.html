Anyone with the app's // Firebase config (which ships inside the app bundle, so it's not really // secret) can technically send requests as if they were any user. // // Given that constraint, these rules can't enforce true per-user // ownership. What they DO meaningfully accomplish: // - Block bulk/listing reads of anything that would expose every user's // sensitive data at once (PIN hashes are isolated in a subcollection // that can only ever be read one document at a time, by exact path — // never listed). // - Reject writes that don't match the expected shape for each // document type, so malformed/junk data can't be written even by an // anonymous request. // - Prevent destructive bulk operations like deleting whole collections. // // A genuinely complete fix would add real Firebase Authentication so // request.auth.uid could be checked against document ownership.
